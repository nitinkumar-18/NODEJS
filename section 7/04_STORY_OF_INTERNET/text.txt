🌐 The Problem in Early Internet (1970s)

In the early days of ARPANET (the precursor to the Internet):

Computers were identified by numeric addresses.
//TELNET COMMAND IN TERMINAL JUST A APPLICATION
Example: 26.0.0.73

Humans found these hard to remember.

👉 There was no Google, no domains, no easy naming system.

🌐 Elizabeth Feinler’s Role

Elizabeth Feinler worked at the Stanford Research Institute (SRI), running the Network Information Center (NIC).

Her team:

✔ Maintained the master directory of all connected computers
✔ Created the HOSTS.TXT file
✔ Assigned human-readable names to computers

Example:

SRI-NIC.ARPA → 26.0.0.73

This was the first step toward DNS.

🌐 Birth of Domain Naming Structure

Feinler’s group introduced the idea of domain suffixes such as:

.com → commercial organizations

.edu → educational institutions

.gov → government

.mil → military

.org → organizations

👉 These categories are still used today.

🌐 From HOSTS.TXT to DNS (1983)

As the network grew:

Manual HOSTS.TXT became impossible to manage.

Engineers created the Domain Name System (DNS) in 1983.

DNS automated name resolution.

👉 Feinler’s naming structure and early directory system inspired DNS.

🌐 Why She Matters

Because of her work:

✅ We use easy names like google.com instead of numbers
✅ The Internet became user-friendly
✅ The domain naming system structure was established

✅ In simple words:

Elizabeth Feinler helped create the first naming system for computers, which later evolved into DNS — the system that turns website names into IP addresses.












🌐 DNS (Domain Name System)

DNS is like the phonebook of the Internet.

👉 It converts website names into IP addresses so computers can find each other.

Example:

google.com  →  142.250.183.206

Humans remember names, computers use numbers.

🧠 Why DNS is Needed

Without DNS, you would need to remember:

142.250.183.206
31.13.71.36
172.217.160.78

Instead we use:

✅ google.com
✅ facebook.com
✅ youtube.com

⚙️ How DNS Works (Step-by-Step)

When you type google.com in your browser:

1️⃣ Browser Cache

Checks if IP is already stored.

2️⃣ DNS Resolver (ISP)

Your Internet provider checks its database.

3️⃣ Root DNS Server

Directs request to the correct domain server.

4️⃣ TLD Server (.com)

Points to the authoritative server.

5️⃣ Authoritative DNS Server

Returns the exact IP address.

6️⃣ Website Loads 🎉

Browser connects to the server using the IP.

🧩 Types of DNS Servers

✔ Recursive Resolver → starts the lookup
✔ Root Server → top-level directory
✔ TLD Server → handles .com, .org, .edu
✔ Authoritative Server → final IP provider

🌍 Example in Real Life

Think of DNS like:

📱 Contact name → Phone number
📍 Place name → GPS coordinates

🧑‍💻 Who Helped Create DNS?

Paul Mockapetris designed DNS in 1983.

Elizabeth J. Feinler created early naming systems that inspired it.

🚀 Advantages of DNS

✅ Easy to remember names
✅ Fast internet navigation
✅ Scalable for billions of websites
✅ Supports email & services

✅ In one line:

👉 DNS converts domain names into IP addresses so browsers can load websites.
