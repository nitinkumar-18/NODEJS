idempotent means same input dene par same output mile














1️⃣ GET
✔ Purpose:

Server se data lena.

Example:
GET /users

👉 users list milegi.

✔ Safe: YES
✔ Idempotent: YES

Same request → same data.

🌍 Real use:

✅ webpage load
✅ products list
✅ search results
✅ API fetch

























2️⃣ POST
✔ Purpose:

naya data create karna.

Example:
POST /users

Body:

{ "name": "Tanish" }

👉 new user create hoga.

❌ Safe: NO
❌ Idempotent: NO

Agar 2 baar bheja → 2 users ban jayenge.

🌍 Real use:

✅ signup
✅ login
✅ payment
✅ form submit















3️⃣ PUT
✔ Purpose:

resource ko update ya replace karna.

Example:
PUT /users/5

👉 user 5 ko replace karega.

✔ Idempotent: YES

Same request 10 baar bhejo → same result.

❌ Safe: NO

server change hota hai.

🌍 Real use:

✅ profile update
✅ settings replace
✅ resource overwrite



















🔹 4️⃣ DELETE
✔ Purpose:

resource delete karna.

DELETE /users/5

👉 user delete.

✔ Idempotent: YES

1 baar delete → ho gaya
10 baar delete → same result

❌ Safe: NO
🌍 Real use:

✅ account delete
✅ post delete
✅ remove item



















5️⃣ HEAD
✔ Purpose:

GET jaisa hi, par sirf headers milte hain.

HEAD /file.zip

👉 file size, last modified milta hai
👉 file download nahi hoti.

✔ Safe: YES
✔ Idempotent: YES
🌍 Real use:

✅ file size check
✅ caching validation
✅ server status check


















6️⃣ OPTIONS
✔ Purpose:

server kya allow karta hai ye batata hai.

OPTIONS /users

Response:

Allow: GET, POST, PUT
✔ Safe: YES
✔ Idempotent: YES
🌍 Real use:

✅ CORS preflight request
✅ API capability check

Browser automatically bhejta hai.













7️⃣ PATCH
✔ Purpose:

partial update.

PATCH /users/5

Body:

{ "name": "Tanish" }

👉 sirf name update.

❌ Safe: NO
❌ Idempotent: NO

Agar multiple patch operations different ho → result change.

🌍 Real use:

✅ profile edit
✅ partial update
✅ toggle status
