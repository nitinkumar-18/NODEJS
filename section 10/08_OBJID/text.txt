JSON Schema is a standard for validating JSON data.

👉 It defines:

Structure (fields)
Data types
Required properties
Constraints (min/max, patterns, etc.)

Think of it like:

“Rules for your JSON data”















JSON Data:
{
  "name": "Tanish",
  "age": 20
}






JSON Schema:
{
  "type": "object",
  "properties": {
    "name": { "type": "string" },
    "age": { "type": "number" }
  },
  "required": ["name", "age"]
}



Possible values:
string
number
integer
boolean
object
array
null



























👉 Validation system = 3 cheezein milke kaam karti hain:

validator → rules define karta hai
validationLevel → kab apply hoga
validationAction → fail hone pe kya hoga




🔹 2. JSON vs JSON Schema (CONFUSION CLEAR)
✅ JSON Data (Actual data stored)
{
  "name": "Tanish",
  "age": 20,
  "email": "tanish@gmail.com"
}

👉 Yeh database mein store hota hai

✅ JSON Schema (Rules)
{
  "bsonType": "object",
  "required": ["name", "email"],
  "properties": {
    "name": { "bsonType": "string" },
    "age": { "bsonType": "int", "minimum": 18 },
    "email": { "bsonType": "string" }
  }
}
